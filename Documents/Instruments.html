BasicSynth Instruments

The BasicSynth library includes a variety of pre-built instruments that can be included in a synthesis system or referenced in the BSynth program. Each instrument is defined by a single class in the Instruments library. All instrument classes inherit from the Instrument base class and thus have the same set of publicly accessible methods.

Instrument configuration may be made programmatically, or loaded from an XML file, allowing a single instrument to be setup for a wide variety of sounds. This document describes the BasicSynth instruments, the XML format, and the parameter numbers for each instrument property.

Instrument configurations are typically stored in an XML file containing multiple instrument definitions. The root tag for the file is instrLib with each instrument configuration contained within the instr tag.

 

<instrLib>

  <instr id="" type="" name="" desc="">

  </instr>

</instrLib>

 

The id attribute is a unique number for the instrument and is used internally to locate instruments. However, the instrument manager will check the value and re-assign a unique number if needed. In general, it is better to locate instruments by name than by number unless you are sure all numbers are unique across all loaded libraries. The type attribute indicates the class used for the instrument and must match a name known to the instrument manager. The name attribute provides a name to locate the instrument. The desc attribute is optional and for information only. The instrument loading code reads and stores the value in the instrument table entry.

The number and IDs for instrument parameters are specific to each instrument, but many parameter types are common to all instruments. The range of values for common parameters are as follows:

 

Frequency -- specified in Hz

Volume/Amplitude -- normalized to a linear amplitude range 0-1

Wavetable Index -- can reference any defined wavetable; pre-defined are 0=sin, 1=saw, 2=square, 3=triangle, 4=pulse

Envelope Rate -- in seconds

Envelope level -- usually 0-1, but may vary depending on application

Envelope type -- 1=linear, 2=exponential 3=log, 4=sustain

Tone Instrument

The Tone instrument contains a single oscillator, ADSR envelope generator and LFO generator. The waveform for the oscillator is selectable, providing for a variety of sounds. Although limited in timber, it is fast and very easy to configure.

The Tone instrument class is defined in Src/Instruments/Tone.h with the implementation in Src/Instruments/Tone.cpp.

The XML tags for the Tone instrument are as follows.

 

<instr id="" type="Tone" name="" desc="">

  <osc frq="" wt="" vol="" />

  <env st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <lfo frq="" wt="" atk="" amp="" />

</instr>

FM Tone Instrument

The FMTone instrument is similar to the Tone instrument, but uses a two-oscillator FM generator in place of the single oscillator. The waveform is not selectable and is always a sine wave. The carrier-modulator ratio and modulation index are set by two parameters.

The FMTone instrument class is defined in Src/Instruments/Tone.h with the implementation in Src/Instruments/Tone.cpp.

FM Synthesis Instrument

The FMSynth instrument incorporates two modulators and one carrier, each with a separate envelope. A filtered noise generator can be mixed with the FM signal to simulate noise transients in the sound. Both the FM signal and noise signal can be fed to an all-pass delay line to add resonance to the sound.

Either of the two modulators can be applied to the carrier. The second modulator can also be applied to the first modulator to produce a "stack" configuration. The selection of modulation is made by setting the algorithm parameter to one of the following values:

 

0 -- Modulator I applied to the carrier, modulator II not used

1 -- Modulator II applied to modulator I, modulator I applied to the carrier

3 -- Modulator I and II both applied to the carrier.

4 -- Modulator II applied to modulator I, modulator I and II both applied to the carrier,.

 

The FMSynth instrument class is defined in the file Src/Instruments/FMSynth.h with the implementation in the Src/Instruments/FMSynth.cpp file.

The XML format for the instrument is shown below.

 

<instr id="" type="FMSynth" name="" desc="" >

  <car mix="" dly="" alg="" st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <mod1 mul="" st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <mod2 mul="" st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <nz mix="" dly="" st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <filt st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <dln mix="" dly="" dec="" />

  <lfo frq="" wt="" rt="" amp="" />

</instr>

Additive Synthesis Instrument

The AddSynth instrument duplicates the tone generator of the Tone instrument and adds the resulting waveforms together. Each generator combines a wavetable oscillator and envelope generator. The number of generators is set by a parameter on the instrument and can be as large as needed. A single LFO generator is shared by all oscillators.

The waveform, frequency, and overall volume for each oscillator may be set individually. When waveforms are set to sine waves, this instrument implements a Fourier series and can be used to re-synthesize sounds from an analyzed sound. It is also possible to use complex waveforms and "morph" between the waveforms over time, implementing a wavetable synthesis system.

The envelope generators are variable multi-segment types allowing up to fifteen segments with or without sustain. Each segment is defined by a rate, ending level and curve type

The AddSynth instrument class is defined in Src/Instruments/AddSynth.h with the implementation in Src/Instruments/AddSynth.cpp.

Because of the variable number of oscillators and envelope generators, the XML format for the AddSynth instrument must implement a hierarchy of nodes with variable number of children at each level. The general structure is shown below.

 

<instr id="" type="AddSynth" name="" desc="" >

  <part pn="0" mul="1" >

    <osc frq="440.0" wt="0" />

    <env segs="2" st="0" son="0">

      <seg sn="0" rt="0.05" lvl="1.0" ty="2" />

      <seg sn="1" rt="0.25" lvl="0.0" ty="2" />

    </env>

  </part>

  <lfo frq="" wt="" atk="" amp="" />

</instr>

 

Because the number of values varies, parameter ID numbers must be calculated based on the partial and envelope segment. The parameter ID numbers are formed as a set of bit fields:

 

PN[6] | SN[4] | VAL[5]

 

PN = (partial number + 1) * 512

SN = (segment number + 1)* 32

 

This layout allows for 63 partials, 15 envelope segments per partial, and 32 parameter values for each partial or envelope. Values of 0 for PN and SN indicate a global instrument parameter index that is used to specify parameters that affect the entire instrument, e.g.  fundamental pitch, master volume, LFO.

Subtractive Synthesis Instrument

The SubSynth class defines an instrument that combines one oscillator, noise source, and low-pass filter. The center frequency of the filter is controlled with an envelope generator. The mixture of oscillator and noise fed to the filter is controlled by the mix parameter. An ADSR envelope shapes the overall amplitude of the filtered signal.

The SubSynth instrument class is defined in Src/Instruments/SubSynth.h with the implementation in Src/Instruments/SubSynth.cpp.

The XML format for the instrument is shown below.

 

<instr id="" type="SubSynth" name="" desc="" >

  <osc frq="" vol="" mix="" fg="" wt="" />

  <egs st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <egf st="" atk="" pk="" dec="" sus="" rel="" end="" ty="" />

  <lfo frq="" wt="" rt="" amp="" />

</instr>

Wave File Playback Instrument

The WFSynth  instrument plays a wave file. The wave file can be played once or looped, and a simple AR envelope may be applied to the file for fade-in and fade-out. In order to efficiently manage multiple files, the instrument loads and maintains a cache of wave files. Each file is assigned an ID number. A file tag in the XML file causes a WAVE file to be loaded into the cache and identified with the given number. The id parameter on the instrument identifies which wave file should be used for playback and can be set dynamically as well as from the XML file. Any file that is to be referenced during playback must be loaded through the XML file. However, all instances of this class share the same cache. Thus, the file can be loaded in any instrument definition in the library and referenced from any instance of the instrument.

The WFSynth class is defined in the file Src/Instruments/WFSynth.h with the implementation in Src/Instruments/WFSynth.cpp.

The XML format for the WFSynth instrument is shown below.

 

<instr id="" type="WFSynth" name="" desc="" >

  <wvf id="" lp="0|1" pa="0|1" />

  <env ar="" rr="" />

  <file name="" id="" >

</instr>

Matrix Synthesis Instrument

The MatrixSynth instrument combines eight oscillators and eight envelope generators in a variable configuration, along with LFO and dynamic panning.

Any oscillator can be designated a signal output, a modulator, or both. Separate parameters determine the amplitude for signal output and modulation output. Oscillator signals are routed using a set of flags on each oscillator. The flags are defined as follows.

Bit 0  - If set, the oscillator is on and output of the oscillator will be processed. If clear, the oscillator is skipped during sample generation.

Bit 1 -- If set, the oscillator output is multiplied by the volume level and added to the audio signal output.

Bit 2 -- If set, the LFO generator is applied to the oscillator.

Bit 3 -- If set, the Pitch Bend function is applied to the oscillator. (Not implemented in version 1.)

Bit 4 -- If set, the output of the oscillator is sent to effects unit 1

Bit 5 -- If set, the output of the oscillator is sent to effects unit 2

Bit 6 -- If set, the output of the oscillator is sent to effects unit 3

Bit 7 -- If set, the output of the oscillator is sent to effects unit 4

Bit 12 -- If set, internal panning is applied to the audio output.

Bit 16-23 -- These bits select modulator inputs. When set, a bit indicates that the output of the associated oscillator (0-8) should be applied to the frequency modulation input of the oscillator.

Bit 24-31 -- Currently unused; reserved for additional generators, up to 16 total.

This setup allows the MatrixSynth instrument to act as an eight operator FM instrument, an eight tone additive synthesis instrument, or any combination of the two.

The eight envelope generators each have three attack segments, sustain and release. Any envelope generator can be applied to any oscillator allowing multiple oscillators to be controlled with a single envelope generator, or, each with a separate envelope.

Each oscillator that is added as a signal output can be independently panned left or right. When internal panning is turned on, the output bypasses normal mixer panning and allows the spatial position of the sound to be dynamically controlled.

Four effects outputs are available for the instrument. Any amount of each output oscillator can be sent to one of these four outputs. This allows dynamic control over effects such as reverb and chorus. All or part of the instrument’s output can be processed as needed. These outputs must be managed by the instrument manager.

The XML format and parameter table for the MatrixSynth instrument are shown below. Each signal generator and envelope generator in the matrix is given a separate node in the tree. The gen node contains attributes for the oscillators and associated values. The env node contains attributes and child nodes for one envelope generator. Both gen and env may be duplicated up to eight times. The envelope segment nodes are duplicated four times with the segment number varying from 0 to 3 representing attack, decay 1, decay 2, release. The level indicates the value at the end of the segment.

 

<instr id="" type="MatrixSynth" name="" desc="" >

  <mat frq="" vol="" />

  <gen gn="" out="" mod="" wt="" mul="" mnx="" vol="" eg="" lfo="" pan="" fx1="" fx2="" fx3="" fx4="" />

  <env en="" st="" sus="">

      <seg sn="" rt="" lv="" ty="" />

  </env>

  <lfo frq="" wt="" rt="" amp="" />

</instr>

 

The out attribute contains the generator output matrix flags. This is encoded as a text string of 1s and 0s from LSB to MSB going left-to-right. See the description of the bits 0-15 given above.

The mod attribute contains the modulator input matrix flags (bits 16-31 of the flags). This is encoded as a text string of 1s and 0s from LSB to MSB going left-to-right. Modulator 0 is the first bit, modulator 1 the second, bit, etc. For example, a value of 0x13 would be "11001000" and would indicate generators 0, 1 and 4 are applied to the oscillator as modulation inputs.

Because of the large number of parameters, parameter IDs are calculated based on the oscillator, envelope and envelope segment number. The upper byte of the ID indicates the generator number. The lower byte indicates the value index. Since we have 8 oscillators and 8 envelope generators, three bits are needed to indicate the generator number. However, to allow for expansion to 16 generators, we reserve four bits. Two bits are used to indicate the type, oscillator or envelope. A value of 0 for the type indicates a global value or one of the reserved event IDs. A value of 1 for the type indicates an oscillator, while a value of 2 indicates an envelope.

 

type[2] | gen[4] | seg[2] | val[6]

 

GN = (oscillator * 256) + 4096

EN = (envelope * 256) + 8192

SN = (segment * 64)
